# Bookmark Navigation 离线 amd64 安装包设计

- 日期：2026-07-14
- 状态：已确认，待实现
- 适用范围：Linux `x86_64/amd64`

## 1. 目标

在已部署的 Linux 服务器上将 Bookmark Navigation 构建为可离线分发的 Docker 安装包。管理员将压缩包上传到另一台服务器后，只需解压并执行一条安装命令，即可完成首次安装或原地升级。

该方案不依赖镜像仓库，支持自定义宿主机监听地址和映射端口，并在重复执行时保留已有数据与账号。

## 2. 非目标

- 不在安装脚本中自动安装 Docker 或修改目标机软件源。
- 不在安装脚本中配置域名、HTTPS 证书或反向代理。
- 不支持 `arm64` 或其他 CPU 架构。
- 本期不提供自动回滚和卸载命令；但在升级前创建数据库备份。

## 3. 交付物

离线包名称使用版本和架构标识，例如：

```text
bookmark-navigation-offline-amd64-20260714-r2.tar.gz
```

解压后目录：

```text
bookmark-navigation-offline-amd64-20260714-r2/
├── install.sh
├── compose.yaml
├── image/
│   └── bookmark-navigation-amd64.tar
├── config/
│   └── bookmark_policy.json
├── docs/
│   └── DEPLOYMENT.md
├── VERSION
├── MANIFEST
└── SHA256SUMS
```

`compose.yaml` 只引用离线包导入的已构建镜像，不包含 `build:`，因此目标机不需要源码、Node.js 或 Python 构建环境。

## 4. 用户接口

基本安装：

```bash
sudo ./install.sh --listen 127.0.0.1 --port 8080
```

直接监听公网或内网网卡：

```bash
sudo ./install.sh --listen 0.0.0.0 --port 18080
```

支持参数：

| 参数 | 默认值 | 说明 |
| --- | --- | --- |
| `--listen ADDRESS` | `127.0.0.1` | 宿主机监听地址 |
| `--port PORT` | `8080` | 宿主机映射端口，范围 `1-65535` |
| `--install-dir PATH` | `/opt/bookmark-navigation` | 运行配置与 Compose 文件目录 |
| `--data-dir PATH` | `/opt/bookmark-navigation/data` | SQLite、备份和运行数据目录 |
| `--username USER` | `admin` | 仅在首次创建账号时使用 |
| `--password-file FILE` | 无 | 从权限受控文件读取初始密码 |
| `--yes` | 关闭 | 跳过非敏感确认，用于自动化 |
| `--help` | - | 显示帮助 |

不支持通过命令行参数直接传入密码，避免密码出现在 shell history 和进程列表中。交互执行且没有提供 `--password-file` 时，脚本使用隐藏输入读取和二次确认密码。

## 5. 首次安装流程

1. 确认以 root 身份运行，或由管理员通过 `sudo` 调用。
2. 验证操作系统为 Linux，架构为 `x86_64/amd64`。
3. 验证 Docker Engine 可用，且 `docker compose` 为 Compose v2。
4. 验证必需文件存在，并根据 `SHA256SUMS` 检查完整性。
5. 校验监听地址、端口、路径和用户名。
6. 检查目标端口是否已被占用。
7. 执行 `docker load` 导入已构建镜像。
8. 创建安装目录、数据目录与运行配置，并设置与容器 UID `10001` 匹配的权限。
9. 启动数据库迁移任务。
10. 在空数据库中创建初始管理员账号。
11. 启动服务，等待容器健康检查通过。
12. 输出本机访问地址、安装目录、数据目录和常用运维命令。

## 6. 重复执行与升级

安装脚本必须幂等。对已安装目录重复执行时：

1. 读取已有安装状态，不重新创建用户，不删除数据。
2. 如新命令显式传入 `--listen` 或 `--port`，更新对应运行配置。
3. 加载安装包中的新镜像。
4. 升级前对 SQLite 数据库创建带时间戳的备份。
5. 执行 Alembic 数据库迁移。
6. 使用新镜像重建容器，但不重建数据目录。
7. 等待健康检查。失败时返回非零状态，显示容器状态、最近日志和备份位置。

如果用户在升级时提供 `--username` 或 `--password-file`，脚本必须明确提示这些参数不会修改已有账号，不可静默重置密码。

## 7. 持久化和运行配置

安装目录保存可重建的运行文件：

```text
/opt/bookmark-navigation/
├── compose.yaml
├── .env
├── config/bookmark_policy.json
├── data/
└── install-state
```

`.env` 由 root 所有且限制读取，包含镜像标签、监听地址、端口、数据目录和 Docker 网络参数，不保存明文密码。

`data/` 作为独立持久化边界，包含 SQLite 数据库、备份和应用运行数据。镜像升级、Compose 文件更新和容器重建都不得覆盖该目录。

## 8. Docker 网络

应用使用独立 bridge 网络和固定网关，以保持应用对反向代理的信任边界。安装脚本检测默认子网是否与现有 Docker 网络冲突；冲突时从预定的私有网段中选择可用子网，并将结果固化到 `.env`。

宿主机端口绑定通过 `${NAV_BIND_ADDRESS}:${NAV_PORT}:8080` 生效。默认仅绑定 `127.0.0.1`，以便在同机反向代理后使用；只有管理员显式指定时才绑定 `0.0.0.0` 或其他地址。

## 9. 错误处理

- 系统、架构、Docker、Compose 或包完整性检查失败时，在变更目标机前终止。
- 端口被占用时终止，并显示冲突端口；不自动停止其他服务。
- 已有安装目录与记录状态不一致时终止，不猜测或覆盖现有内容。
- 数据库迁移失败时不启动新容器，保留升级前备份。
- 健康检查超时时输出 `docker compose ps` 和最近容器日志，并以非零状态退出。

## 10. 安全边界

- 安装包通过 `SHA256SUMS` 检查传输完整性；SHA-256 不等于来源签名，包本身仍应通过可信渠道传递。
- 初始密码只从 TTY 隐藏输入或权限受控的密码文件读取。
- 脚本不将密码写入 `.env`、日志或安装状态。
- 运行容器保持非 root UID `10001`。
- 默认绑定回环地址，对外暴露必须由管理员显式指定。

## 11. 测试与验收

实现应包含自动化脚本测试和服务器端验收，至少覆盖：

- 参数默认值、自定义监听地址和自定义端口。
- 无效端口、不支持架构、缺少 Docker/Compose 和校验失败。
- 首次安装创建数据和管理员。
- 重复执行不重置账号、不丢失数据。
- 更换镜像后迁移成功，并产生升级前备份。
- 指定新端口重复执行后，新绑定生效。
- 应用健康检查、登录页和管理员登录成功。
- 目标机在无源码、无 Node.js/Python 构建环境、无镜像仓库访问的情况下可完成安装。
