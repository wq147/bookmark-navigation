# Security Policy

## 报告安全问题

请不要在公开 Issue 中提交尚未修复的漏洞、密码、Token、私钥、真实主机信息、数据库或私人书签内容。

在 GitHub 仓库启用 **Private vulnerability reporting** 后，请优先通过仓库的 **Security → Report a vulnerability** 私下报告。报告应包含受影响版本、复现条件、预期影响和可安全共享的最小复现；请对日志和截图做脱敏处理。

在私密报告渠道尚未启用时，请暂缓公开披露，并通过仓库所有者在 GitHub 个人资料中公开的安全联系方式沟通。本项目不在文档中虚构或嵌入未经确认的邮箱地址。

## 支持范围

项目尚未发布稳定版本。安全修复以默认分支的最新源码为准；历史离线包只有在对应发布说明明确标记受支持时才获得安全更新。

## 部署责任

- 使用强密码并保护 `.env`、SQLite 数据、备份和初始密码文件。
- 默认仅绑定 `127.0.0.1`，公网访问应置于 HTTPS 反向代理之后。
- 不要把可信代理范围设置为 `*`，也不要让多个 Uvicorn worker 共享同一个 SQLite 数据目录。
- 公开网络服务的修改版本还应履行项目 `AGPL-3.0-only` 许可证规定的源代码提供义务。
